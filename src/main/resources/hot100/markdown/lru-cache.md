# LRU 缓存

## 题目摘要
设计最近最少使用缓存，要求 `get/put` 平均 O(1)。

## 核心思路
哈希表定位节点 + 双向链表维护访问顺序（头新尾旧）。

## 为什么这样做
哈希表负责快查，链表负责快删快插，组合后满足 O(1) 目标。

## 常见错误
- 节点移动时只改一侧指针。
- 容量满淘汰后忘记同步删除哈希表项。

## 复杂度分析
`get/put` 平均 O(1)，空间 O(capacity)。
